## Delivery Workflow

### Day 0: Sale Closed
```
1. Payment received (Stripe)
2. Onboarding form auto-sent
3. Slack notification to delivery team
```

### Day 1: Setup
```
1. Review onboarding form
2. Provision n8n workflows (template + customize)
3. Request OAuth connections (email to client)
4. Client connects accounts
```

### Day 2: Testing
```
1. Run test scenarios
2. Validate all connections
3. Fix any issues
4. Document client-specific configs
```

### Day 3: Launch
```
1. Enable live mode
2. Send launch email to client
3. Monitor first 24h closely
4. First check-in call (optional)
```

### Week 1: Stabilize
```
1. Daily monitoring
2. Fix any edge cases
3. Collect feedback
4. Optimize based on data
```

### Ongoing: Maintain
```
1. Weekly performance reports
2. Monthly check-in (optional)
3. Issue response per SLA
4. Quarterly optimization review
```

## Sales → Delivery Handoff

### What Sales Collects:
```
- Signed contract
- Payment method
- Primary contact email/phone
- Timezone
- Urgency level
```

### What Delivery Needs:
```
- Completed onboarding form
- OAuth connections done
- Any passwords/API keys (secure vault)
- Specific requirements/exceptions
```

### Handoff Trigger:
```
Payment confirmed → Zapier/n8n → 
  1. Create client record
  2. Send onboarding form
  3. Create Slack channel
  4. Add to delivery queue
```
