## SCRIPT 10 — WHAT IS IDEMPOTENCY

🥴 MORTY: RICK, I ACCIDENTALLY SENT THE SAME PAYMENT REQUEST TWICE. DID I JUST CHARGE SOMEONE DOUBLE?
🧪 RICK: THAT DEPENDS ON WHETHER THE API IS IDEMPOTENT.
🥴 MORTY: IDEMPO-WHAT?
🧪 RICK: IDEMPOTENT. SENDING THE SAME REQUEST MULTIPLE TIMES HAS THE SAME EFFECT AS SENDING IT ONCE.
🥴 MORTY: SO A DOUBLE PAYMENT REQUEST WOULD ONLY CHARGE THEM ONCE?
🧪 RICK: IF THE API IS BUILT CORRECTLY, YES. YOU SEND AN IDEMPOTENCY KEY — A UNIQUE ID — AND THE SERVER KNOWS IT ALREADY PROCESSED THAT REQUEST.
🥴 MORTY: AW GEEZ, THAT'S SO IMPORTANT.
🧪 RICK: NETWORK ERRORS HAPPEN. RETRIES HAPPEN. WITHOUT IDEMPOTENCY, A RETRY MEANS A DUPLICATE. THAT'S A DISASTER.
🥴 MORTY: SO THIS IS HOW STRIPE DOESN'T DOUBLE-CHARGE PEOPLE?
🧪 RICK: EXACTLY HOW. IT'S NOT MAGIC. IT'S ENGINEERING.
🧪 RICK: LOCK IN AND FOLLOW FOR MORE ON THE API PATTERNS THAT KEEP REAL MONEY SAFE.

[Loops back to: Morty's panic about accidentally sending a duplicate payment request]
