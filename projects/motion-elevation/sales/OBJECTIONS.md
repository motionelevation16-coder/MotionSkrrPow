# Objection Handling Playbook 🛡️

*Every doubt, answered. Every hesitation, resolved.*

1. **In cold outreach** → Preempt objections in follow-up emails

## Price Objections

### "It's too expensive"

**Reality Check:** They're comparing to $0, not to the alternative.

**Response:**
```
I get it—it's an investment. 

Quick question: How many hours per week do you (or your team) spend 
on the tasks we'd automate?

[They answer, e.g., "10-15 hours"]

At even $25/hour, that's $1,000-1,500/month in labor. 
Our automation costs $297-697 and works 24/7, no sick days.

The question isn't "can I afford this?"—it's "can I afford NOT to?"
```

**If they push back:**
```
What if we start with one automation? You see the ROI, then expand.
No risk, just proof.
```

---

### "I can't afford it right now"

**Reality Check:** Timing objection, not value objection.

**Response:**
```
Totally understand cash flow concerns.

Here's an option: We start with a single automation at $297/month.
If it saves you 10+ hours/month (it will), you've already made money.

Then we scale when you're ready.

Would that work?
```

**Alternative:**
```
What if we tie payment to results? First month at 50% off, 
you only continue if you see ROI.
```

---

### "I'll just do it myself / use Zapier"

**Reality Check:** They overestimate their time and underestimate complexity.

**Response:**
```
Totally fair—Zapier's a great tool.

Here's what usually happens: People spend 20 hours building something 
that kind of works, then another 10 hours fixing it when it breaks.

We've built 50+ of these workflows. We know the edge cases, 
the API quirks, the failure points.

You could do it yourself—or you could have it running in 48 hours 
while you focus on what you're actually good at.

Which sounds better?
```

## Trust Objections

### "How do I know this will work?"

**Response:**
```
Great question—you shouldn't just trust me.

Here's what we do:
1. Free audit call—we show you exactly what we'd automate
2. You see the workflow before we build it
3. We test everything in sandbox before going live
4. First 30 days: if it's not working, we fix it or refund you

There's no risk on your side.
```

---

### "I've been burned by agencies before"

**Reality Check:** Trust wound. Be empathetic, not defensive.

**Response:**
```
Yeah, I hear that a lot. Most agencies overpromise and underdeliver.

Here's how we're different:
- We build specific automations, not vague "strategies"
- You see results in days, not months
- We document everything—you own the workflows
- No lock-in contracts

If you want, I can show you something we built for [similar business]. 
You'll see exactly what you're getting.
```

---

### "I need to think about it"

**Reality Check:** Usually means they're not sold yet.

**Response (Soft):**
```
Of course—take your time.

Quick question so I can help you think: 
What specifically are you weighing? Is it the price, the timing, 
or whether automation fits your business?

[Wait for answer, then address that specific concern]
```

**Response (Direct):**
```
Totally get it. Just so I understand—is there something specific 
holding you back? Or is it just not the right time?

If it's a specific concern, I'd rather address it now than have you 
wondering later.
```

---

### "I need to talk to my partner/team"

**Reality Check:** Real concern. Don't push too hard.

**Response:**
```
Makes sense—big decisions need input.

Would it help if I put together a one-pager you can share with them? 
Covers exactly what we'd build, the cost, and expected ROI.

Then we can hop on a quick call with everyone if they have questions.
```

## Technical Objections

### "I don't understand how this works"

**Response:**
```
Let me make it super simple:

1. You tell us what tasks eat your time
2. We build a system that does those tasks automatically
3. You approve it, we turn it on
4. It runs in the background, you check results

You don't need to touch code, APIs, or anything technical. 
That's our job.

Think of it like hiring a robot employee—one that never sleeps 
and never makes mistakes.
```

---

### "What if something breaks?"

**Response:**
```
Great question—things do occasionally need tweaking.

Here's how we handle it:
- All workflows have error monitoring built in
- If something fails, we get notified immediately
- Monthly maintenance is included in your plan
- Critical issues: we respond within 24 hours (often same-day)

You're not on your own. We babysit the system so you don't have to.
```

---

### "My business is too unique/complicated"

**Response:**
```
That's actually pretty common—every business thinks they're special.
(And in some ways, you are!)

But here's the thing: the underlying problems are usually the same.
- Lead follow-up
- Repetitive emails
- Data entry between tools
- Scheduling back-and-forth

We've automated businesses in [list 2-3 similar niches]. 
The surface looks different, but the structure is similar.

Let me ask: what's the ONE task that eats the most time in your business?
[Then show how you'd automate that specific thing]
```

---

### "I use [obscure tool]. Can you integrate that?"

**Response (If Yes):**
```
Yep, we work with [tool name]. We've integrated it with 
[other common tools] before—should be straightforward.
```

**Response (If Unsure):**
```
I'm not 100% sure on that one—let me check if there's 
an API or Zapier connection.

If there's no direct integration, we can usually work around it 
with webhooks or scheduled data syncs.

Can you send me a link to the tool? I'll confirm within 24 hours.
```

**Response (If No Integration Exists):**
```
That tool doesn't have an API we can connect to directly.

But here's what we can do:
- Export/import via CSV (scheduled)
- Email parsing if it sends notifications
- Manual bridge step if volume is low

Would one of those work for your use case?
```

## Timing Objections

### "I'm too busy to set this up right now"

**Reality Check:** Ironic—they're too busy doing the work automation would eliminate.

**Response:**
```
I hear that a lot—and honestly, it's the exact reason to do this now.

Here's the thing: We do 95% of the work. 
Your only job is a 30-min kickoff call and approving what we build.

We literally automate so you're LESS busy.

When's a 30-minute window this week or next?
```

---

### "Maybe in a few months"

**Response:**
```
Totally understand. Out of curiosity—what changes in a few months 
that would make it a better time?

[Listen to answer]

Here's the thing: Every month you wait is another month of 
manual work. The sooner you start, the sooner you get those hours back.

What if we just did the audit call now—no commitment—so when 
you're ready, you already know exactly what we'd build?
```

---

### "I'm in the middle of [other project]"

**Response:**
```
Got it—focus is important. When does that wrap up?

[They give a date]

Perfect. Let me drop you a note then. In the meantime, 
I'll send over some examples of what we could do for your business.

If anything, automation might actually help you finish that 
project faster—less admin on your plate.
```

## Competition Objections

### "I'm already using [competitor/similar service]"

**Response:**
```
Nice—glad you're already investing in automation.

Quick question: Are you happy with the results, or is there 
something that's not quite working?

[If happy]: Great, sounds like you're covered.
[If not happy]: What's falling short?
[Then address that specific gap]
```

---

### "My VA handles this already"

**Response:**
```
VAs are great for tasks that need human judgment.

But for repetitive stuff—sending the same emails, copying data, 
posting to social—automation is faster, cheaper, and doesn't need 
time off.

Most of our clients keep their VA for high-touch work and 
automate the grunt work. The VA becomes more effective, 
not replaced.

Does that sound useful?
```

2. **Buy time.** Say: "I want to give you a proper answer. Can I get back to you by [tomorrow/end of day]?"

## Objection Prevention (Preemptive Strikes)

### In Your Emails

Add lines like:
```
"No tech skills needed—we handle everything."
(Prevents: "I'm not technical")

"Most clients are live within 48 hours."
(Prevents: "I don't have time to set this up")

"30-day guarantee—if it doesn't work, you don't pay."
(Prevents: "What if it doesn't work?")

"You own everything we build—no lock-in."
(Prevents: "What if I want to leave?")
```

### On Your Landing Page

Include an FAQ section that answers the top 5 objections:
1. What if I'm not technical?
2. How long does setup take?
3. What happens if something breaks?
4. Do I own the automations?
5. What's the ROI?
