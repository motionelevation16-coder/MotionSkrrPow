# 2025-02-25

## OpenClaw Model Prefix Bug

**Issue:** OpenClaw keeps adding `anthropic/` prefix to model names internally, then failing with "Unknown model: anthropic/claude-sonnet-4-6" (or haiku, opus).

**Investigation findings:**
- Config file (`openclaw.json`) is **correct** — models defined without prefix: `claude-sonnet-4-6`, `claude-opus-4-5`, `claude-haiku-3-5`
- Sessions file also stores model without prefix
- But runtime injects `model=anthropic/claude-opus-4-5` and `default_model=anthropic/claude-sonnet-4-6`
- Cron jobs fail because OpenClaw adds prefix before API call

**Root cause:** OpenClaw internal bug — adds `anthropic/` prefix somewhere in code before API calls

**Affected:**
- Cron job "Lyubo Lunch Update" failing with consecutive errors
- Model switching commands

**Solution:** Update OpenClaw
- Installed: 2026.2.9
- Latest: 2026.2.24 (15 versions behind)
- Lyubo approved update but was at work, waiting for confirmation
